## 4. Часть A: модели и обучение (C1-C4)

Опишите коротко и сопоставимо:

- C1 (simple-cnn-base): простая CNN без аугментаций. Использовались базовые transforms и обучение с нуля. Этот эксперимент нужен как baseline для сравнения с аугментациями и transfer learning.
- C2 (simple-cnn-aug): та же архитектура простой CNN, что и в C1, но с augmentation transforms на train. Это позволяет отдельно посмотреть, помогает ли расширение обучающего распределения без изменения самой архитектуры.
- C3 (resnet18-head-only): использовалась pretrained ResNet18. Backbone был заморожен, обучалась только классификационная голова (fc). Эксперимент показывает, насколько полезны уже готовые визуальные признаки без fine-tuning глубоких слоёв.
- C4 (resnet18-finetune): использовалась pretrained ResNet18 с partial fine-tuning. В моём случае дообучались layer4 + fc, то есть верхняя часть сети адаптировалась под конкретный датасет. Этот эксперимент оказался лучшим по качеству.

Дополнительно:

- Loss: CrossEntropyLoss

- Epochs (макс): для C1 и C2 использовалось 5 эпох, для C3 (head-only) — 3 эпохи, для C4 (partial fine-tuning) — 2 эпохи.

## 5. Часть B: постановка задачи и режимы оценки (V1-V2)

### Если выбран detection track

- Модель: fasterrcnn_mobilenet_v3_large_fpn
- V1: `score_threshold = 0.3`
- V2: `score_threshold = 0.7`
- Как считался IoU: IoU считался между predicted bbox и ground truth bbox в формате (x1, y1, x2, y2) как отношение площади пересечения к площади объединения.
- Как считались precision / recall: Сначала выполнялось базовое сопоставление prediction ↔ ground truth при IoU >= 0.5 с matching по убыванию score. 
    После этого считались:
    TP — если bbox сопоставлен с ещё не занятым ground truth при IoU >= 0.5
    FP — если предсказание не нашло подходящий ground truth
    FN — если ground truth остался несопоставленным
    Далее:
    precision = TP / (TP + FP)
    recall = TP / (TP + FN)

## 7. Анализ

Простая CNN ведёт себя на `Flowers102` слабо, потому что датасет достаточно сложный: много классов, похожие визуальные структуры, разнообразные фоны и условия съёмки. Для такого набора данных небольшая CNN, обучаемая с нуля, не успевает выучить достаточно сильные признаки за ограниченное число эпох. В моих результатах аугментации не дали устойчивого улучшения: `C2` оказался хуже `C1`. Это можно объяснить тем, что сама архитектура CNN достаточно ограничена, а выбранные аугментации при таком setup не компенсируют нехватку мощности модели.

Pretrained `ResNet18` помогла очень заметно: даже head-only режим (`C3`) дал большой прирост по сравнению с `C1/C2`, потому что backbone уже содержит сильные визуальные признаки, обученные на большом датасете. Partial fine-tuning (`C4`) оказался лучше, чем head-only, потому что верхняя часть backbone смогла адаптироваться именно под задачу классификации цветов. То есть в моих результатах pretrained признаки полезны сами по себе, но ещё лучше работает их дополнительная адаптация.

Для второй части detection-метрика выбрана корректно: `precision, recall и mean_iou` хорошо подходят для базовой оценки quality detection pipeline без необходимости считать полный mAP. При переходе от `V1` к `V2` произошло ожидаемое изменение: увеличение порога уверенности уменьшило количество предсказаний, поэтому precision вырос, а recall снизился. Это типичное поведение detection-модели. Наиболее показательными ошибками оказались либо лишние предсказания на более низком пороге, либо пропуски объектов на более высоком пороге.



## 8. Итоговый вывод

Как базовый конфиг классификации я бы выбрала `C4`, потому что он дал лучшую `val_accuracy` и лучшую итоговую `test_accuracy`. 

Главное про transfer learning: pretrained backbone даёт сильный старт, а partial fine-tuning позволяет ещё лучше адаптировать модель под конкретный датасет. 

По второй части я поняла, что для detection важен не только сам запуск модели, но и корректная интерпретация метрик: изменение `score_threshold` напрямую влияет на баланс между precision и recall.
